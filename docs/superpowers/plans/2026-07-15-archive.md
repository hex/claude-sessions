# Session Archive Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A reversible "done for now" state: `cs -archive <name>` drops a tracked `.cs/archived` marker that hides the session from the TUI, `cs -list`, and `cs -search` by default; opening the session auto-unarchives it.

**Architecture:** Presence of `$session_dir/.cs/archived` IS the state (content is one advisory line). A new bash fragment `lib/59-archive.sh` owns the verbs and the `_session_is_archived` predicate; `list_sessions`, `search_sessions`, and `launch_claude_code` consult it. The TUI reads the marker into `Session.archived`, hides archived rows at the `apply_filter_and_sort` choke point, and `A` toggles visibility with dimmed rendering.

**Tech Stack:** bash 3.2 + BSD userland (lib/ fragments assembled into bin/cs by build.sh), Rust/ratatui 0.29 (tui/).

**Spec:** docs/superpowers/specs/2026-07-15-archive-design.md

## Global Constraints

- `bin/cs` is GENERATED. Edit `lib/` fragments, run `./build.sh`, and commit the regenerated `bin/cs` in the SAME commit (CI fails on drift). Tests execute `bin/cs`, so build before running them.
- bash 3.2 + BSD userland only: no `local -A`, no `mapfile`, BSD `awk`/`sed`/`stat`/`date`. awk exit codes: a body `exit N` still runs END and END's exit wins — always use the flag+END pattern for boolean awk.
- Every assertion in a bash test function needs `|| return 1` (the harness's `run_test` suppresses errexit).
- Dispatch case arms in `lib/99-main.sh` site A use exactly 8-space indent (tests/test_completions.sh awk-parses them); every new verb must appear in BOTH `completions/_cs` and `completions/cs.bash`.
- `show_help` (lib/10-help.sh) is an unquoted heredoc — NO backticks in help text.
- Marker file: `$session_dir/.cs/archived`, TRACKED (not under `.cs/local/`). One line `archived: <YYYY-MM-DD> by <actor>`. Presence = archived. cs never commits the change.
- Error/info helpers: `error "msg"` prints to stderr and exits 1; `info "msg"` prints green to stdout (lib/05-term.sh:59,64).
- Never call a function that may `error` inside command substitution — `error`'s exit dies in the subshell. Resolution helpers set globals instead (the `cs_split_worktree_name` precedent).
- `cargo test` must run from inside `tui/` (the `.cargo/config.toml` single-thread setting is cwd-scoped).
- Idempotency: archiving an archived session / unarchiving a plain one prints `info` and exits 0.
- Full gates before finishing any task: `bash tests/run_all.sh` and (for TUI tasks) `cd tui && cargo test`.

```bash
# ABOUTME: cs -archive / cs -unarchive: the tracked .cs/archived marker that
# ABOUTME: hides finished sessions from default listings until reopened.

# Resolve a session name for the archive verbs. Sets ARCHIVE_DIR (real path,
# adopted-session symlinks followed) and ARCHIVE_MARKER; errors on an unknown
# session. Called directly, never in command substitution — error must exit
# the caller, not a subshell.
_archive_resolve() {  # name
    ARCHIVE_DIR="$SESSIONS_ROOT/$1"
    if [ ! -d "$ARCHIVE_DIR" ] && [ ! -L "$ARCHIVE_DIR" ]; then
        error "No such session: $1"
    fi
    [ -L "$ARCHIVE_DIR" ] && ARCHIVE_DIR="$(_resolve_symlink_dir "$ARCHIVE_DIR")"
    ARCHIVE_MARKER="$ARCHIVE_DIR/.cs/archived"
}

# True when a session directory carries the archived marker. Presence is the
# whole state; the file's content is advisory.
_session_is_archived() {  # session_dir
    [ -f "$1/.cs/archived" ]
}

run_archive() {
    local name="" force="" arg
    for arg in "$@"; do
        case "$arg" in
            --force|-f) force="true" ;;
            -*) error "Unknown archive option: $arg. Usage: cs -archive <name> [--force]" ;;
            *)
                [ -z "$name" ] || error "Usage: cs -archive <name> [--force]"
                name="$arg"
                ;;
        esac
    done
    [ -n "$name" ] || error "Usage: cs -archive <name> [--force]"
    _archive_resolve "$name"
    if [ -f "$ARCHIVE_MARKER" ]; then
        info "Already archived: $name"
        return 0
    fi
    if [ -z "$force" ] && session_is_live "$ARCHIVE_DIR/.cs"; then
        error "Session '$name' is live (pid $(read_lock_pid "$ARCHIVE_DIR/.cs")); use --force to archive anyway"
    fi
    printf 'archived: %s by %s\n' "$(date +%Y-%m-%d)" "$(cs_actor_slug)" > "$ARCHIVE_MARKER"
    info "Archived: $name (opening it restores it; or run 'cs -unarchive $name')"
}

run_unarchive() {
    local name="${1:-}"
    [ -n "$name" ] || error "Usage: cs -unarchive <name>"
    _archive_resolve "$name"
    if [ ! -f "$ARCHIVE_MARKER" ]; then
        info "Not archived: $name"
        return 0
    fi
    rm -f "$ARCHIVE_MARKER"
    info "Unarchived: $name"
}
```

list_sessions() {
    local tag_filter="" archived_only=""
    while [ $# -gt 0 ]; do
        case "$1" in
            --tag)
                shift
                [ -n "${1:-}" ] || error "Usage: cs -list [--archived] [--tag <tag>]"
                # Stored tags are always lowercase (cs -tag add lowercases on
                # write); lowercase the filter too so it matches regardless
                # of case, mirroring the TUI's parse_tag_query.
                tag_filter=$(printf '%s' "$1" | tr '[:upper:]' '[:lower:]')
                shift
                ;;
            --archived)
                archived_only="true"
                shift
                ;;
            *) error "Unknown list option: $1. Usage: cs -list [--archived] [--tag <tag>]" ;;
        esac
    done

    if [ ! -d "$SESSIONS_ROOT" ]; then
        info "No sessions found"
        return 0
    fi

    local sessions=()
    local hidden_archived=0
    while IFS= read -r -d '' dir; do
        is_session_dir "$dir" || continue
        if [ -n "$tag_filter" ]; then
            _tags_read "$dir/.cs/README.md" | grep -Fqx "$tag_filter" || continue
        fi
        if [ -n "$archived_only" ]; then
            _session_is_archived "$dir" || continue
        elif _session_is_archived "$dir"; then
            hidden_archived=$((hidden_archived + 1))
            continue
        fi
        sessions+=("$(basename "$dir")")
    done < <(find "$SESSIONS_ROOT" -mindepth 1 -maxdepth 1 \( -type d -o -type l \) -print0 | sort -z)

    if [ ${#sessions[@]} -eq 0 ]; then
        info "No sessions found"
        _list_archived_trailer "$hidden_archived"
        return 0
    fi
